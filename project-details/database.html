SQL Database for University

This project involves designing and implementing a relational SQL database to manage various aspects of a university's operations, including students, courses, and faculty.

Technologies Used

  • SQL (MySQL)
  • Database Design Tools (e.g., ERD tools like Lucidchart)
  • Python (for testing and querying)

Challenges Faced

- Designing a normalized database schema to avoid redundancy.
- Writing complex SQL queries to handle multiple table joins.
- Ensuring data integrity and implementing proper constraints.

Outcomes

- Successfully created a scalable and efficient database schema.
- Demonstrated the ability to perform CRUD operations.
- Implemented advanced queries to extract meaningful insights (e.g., student performance reports, course popularity).

Schema Overview

Key Features

  • Student Management: Tracks student information, including personal details and enrolled courses.
  • Course Management: Handles course details, schedules, and instructor assignments.
  • Grading System: Stores grades and generates performance reports.

Sample Queries

-- Retrieve all students enrolled in a specific course:
SELECT students.name, courses.course_name
FROM students
JOIN enrollments ON students.student_id = enrollments.student_id
JOIN courses ON enrollments.course_id = courses.course_id
WHERE courses.course_name = 'Database Systems';

-- Calculate average grade for a course:
SELECT courses.course_name, AVG(grades.grade) AS average_grade
FROM grades
JOIN courses ON grades.course_id = courses.course_id
GROUP BY courses.course_name;
